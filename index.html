=Cuestionario=

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

La diferencia radica en su función, ya que el Internet es la red que conecta a los dispositivos, mientras que la World Wide Web es una de las aplicaciones que permite acceder a la información en esa red.

Internet: Es una red global de computadoras interconectadas que permite el intercambio de información entre dispositivos.

World Wide Web: Es un sistema de información que utiliza Internet como medio para acceder a recursos, como páginas web enlazadas mediante hipervínculos a través de protocolos como HTTP.

En resumen, el Internet es la infraestructura donde vive la WWW.

Diferenciador. (s.f.). Diferencia entre internet y web. Recuperado el 15 de febrero de 2025, de https://www.diferenciador.com/diferencia-entre-internet-y-web/

2. ¿Cuáles son las partes de un URL?

  • Esquema (Protocolo): Define el protocolo de comunicación utilizado, como HTTP o HTTPS.
  • Subdominio (Opcional): Puede incluir prefijos como www o personalizados (blog.ejemplo.com).
  • Dominio: Es el nombre del sitio web, como ejemplo.com.
  • TLD (Top-Level Domain): Es la extensión del dominio, como .com, .org, .net.
  • Puerto (Opcional): Un número que indica el puerto de conexión al servidor, como :80 para HTTP o :443 para HTTPS.
  • Ruta: Especifica la ubicación del recurso dentro del servidor.
  • Parámetros (Query String "?"): Contienen información adicional en formato clave=valor, separados por &.
  • Fragmento (Anchor "#"): Indica una sección específica de la página.
MDN Web Docs. (s.f.). URL - Uniform Resource Locator. Mozilla. Recuperado el 15 de febrero de 2025, de https://developer.mozilla.org/es/docs/Learn/Common_questions/What_is_a_URL

3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Método Propósito Ejemplo
GET Solicitar datos de un recurso en el servidor sin modificarlo. Ejemplo: GET /usuarios
HEADSimilar a GET, pero solo obtiene los encabezados de la respuesta sin el cuerpo.Ejemplo: HEAD /imagen.jpg
POSTEnviar datos al servidor para crear un nuevo recurso.Ejemplo: POST /usuarios
PUTActualizar completamente un recurso en el servidor, reemplazando su contenido.Ejemplo: PUT /usuarios/123
PATCHModificar parcialmente un recurso en el servidor.Ejemplo: PATCH /usuarios/123
DELETEEliminar un recurso del servidor.Ejemplo: DELETE /usuarios/123
MDN Web Docs. (s.f.). Métodos de petición HTTP. Mozilla. Recuperado el 15 de febrero de 2025, de https://developer.mozilla.org/es/docs/Web/HTTP/Methods

4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El método HTTP que se debe utilizar al enviar un formulario HTML con algún usuario y contraseña es POST.

¿Por qué?

  • Seguridad: POST envía los datos en el cuerpo de la solicitud HTTP, lo que evita que la información sensible (como contraseñas) aparezca en la URL, a diferencia de GET.
  • No almacenado en caché ni en historial: A diferencia de GET, los datos enviados con POST no quedan registrados en el historial del navegador ni en los registros del servidor.
  • Mayor capacidad de datos: POST permite enviar grandes cantidades de información, mientras que GET tiene limitaciones en la longitud de la URL.
MDN Web Docs. (s.f.). Usando formularios en HTML. Mozilla. Recuperado el 15 de febrero de 2025, de https://developer.mozilla.org/es/docs/Learn/Forms/Sending_and_retrieving_form_data

5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Cuando se accede a una página web a través de un URL en un navegador, se utiliza el método HTTP GET.

¿Por qué?

  • Recuperación de recursos:
    • Se usa para solicitar datos de un servidor sin modificar nada en él.
  • Datos en la URL:
    • GET puede incluir parámetros en la URL.
  • Almacenamiento en caché :
    • Las respuestas a GET pueden ser almacenadas en caché por los navegadores.
MDN Web Docs. (s.f.). Métodos de petición HTTP. Mozilla. Recuperado el 15 de febrero de 2025, de https://developer.mozilla.org/es/docs/Web/HTTP/Methods/GET

6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

El código de estado HTTP 200 "OK" indica que la solicitud del cliente fue procesada correctamente por el servidor y que la respuesta contiene el recurso solicitado, así que el servidor devuelve el contenido solicitado.

Es el código de estado más común para respuestas GET aunque también puede aplicarse a POST, PUT, etc.

MDN Web Docs. (s.f.). Códigos de estado de respuesta HTTP. Mozilla. Recuperado el 15 de febrero de 2025, de https://developer.mozilla.org/es/docs/Web/HTTP/Status

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Depende de la causa del error 404 "Not Found", ya que este error significa que el recurso al que el usuario está tratando de acceder no existe o no fue encontrado en el servidor y no siempre es culpa del desarrollador. Incluso puede ser error del usuario al no escribir correctamente la URL, la conexión a internet del usuario falló, etc.

MDN Web Docs. (s.f.). Errores HTTP 404 Qué son y cómo corregirlos. Mozilla. Recuperado el 15 de febrero de 2025, dehttps://developer.mozilla.org/es/docs/Web/HTTP/Status/404

8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, ya que el error 500 "Internal Server Error" significa que hay un problema interno con el servidor que impide que la solicitud se complete de forma correcta. Esto puede ser causado por errores en el código del backend, errores en la configuración del servidor, errores de permisos, errores con las APIs o errores en la base de datos.

MDN Web Docs. (s.f.). 500 Internal Server Error. Mozilla. Recuperado el 15 de febrero de 2025, de https://developer.mozilla.org/es/docs/Web/HTTP/Status/500

9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Cuando un atributo o elemento en HTML5 está depreciado (deprecated), significa que su uso ya no es recomendado y que podría ser eliminado en futuras versiones del estándar. Aunque aún puede funcionar en algunos navegadores, su compatibilidad no está garantizada y se recomienda reemplazarlo con alternativas más modernas.

La deprecación ocurre cuando un elemento se considera obsoleto o innecesario debido a mejoras en el lenguaje, como el uso de CSS, nuevas etiquetas semánticas o APIs avanzadas.

MDN Web Docs. (s.f.). Elementos y atributos obsoletos en HTML5. Mozilla. Recuperado el 15 de febrero de 2025, dehttps://developer.mozilla.org/es/docs/Web/HTML/Obsolete_Elements

10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 es la evolución de HTML 4 pues ofrece una estructura más limpia, soporte nativo para multimedia, mejores formularios, nuevas APIs y mayor compatibilidad con dispositivos modernos, mejorando la funcionalidad, estructura y compatibilidad con nuevas tecnologías web.

Soporte para contenido multimedia

  • HTML4: Requería complementos externos como Flash o Silverlight para reproducir audio y video.
  • HTML5: Introduce las etiquetas <audio> y <video>, permitiendo reproducir contenido sin complementos adicionales.

Nuevas etiquetas semánticas

  • HTML4: Utilizaba principalmente <div> y <span> para estructurar contenido.
  • HTML5: Introduce etiquetas semánticas como <header>, <article>, <section>, <nav> y <footer>, mejorando la accesibilidad y la estructura del contenido.

Mejor soporte para aplicaciones web

  • HTML4: No incluía APIs integradas para desarrollo web avanzado.
  • HTML5: Incluye APIs como:
    • Canvas API: Para gráficos y animaciones 2D.
    • Web Storage (LocalStorage y SessionStorage): Para almacenar datos en el navegador sin cookies.
    • Geolocation API: Para obtener la ubicación del usuario.

Eliminación de etiquetas obsoletas

  • HTML4: Utilizaba etiquetas y atributos desactualizados como <font>, <center> y <bgcolor>.
  • HTML5: Elimina estas etiquetas en favor de CSS para mejorar la presentación.

Mejoras en formularios y validación de datos

  • HTML5 introduce nuevos atributos y tipos de entrada como email, date, number y placeholder, mejorando la validación sin necesidad de JavaScript.

Compatibilidad con dispositivos móviles

  • Mejora la compatibilidad con móviles gracias a la etiqueta <meta viewport>.
MDN Web Docs. (s.f.). Diferencias entre HTML4 y HTML5. Mozilla. Recuperado el 15 de febrero de 2025, de https://developer.mozilla.org/en-US/docs/Glossary/HTML5

11. ¿Qué componentes de estructura y estilo tiene una tabla?

  • <table> → Elemento que define la tabla.
  • <tr> (Table Row) → Define una fila en la tabla.
  • <th> (Table Header) → Define una celda de encabezado (negrita y centrada por defecto).
  • <td> (Table Data) → Define una celda de datos.
  • <caption> → Agrega un título o descripción a la tabla.
  • <thead> → Agrupa los encabezados de la tabla.
  • <tbody> → Contiene el cuerpo de la tabla.
  • <tfoot> → Define una sección al final de la tabla, útil para resúmenes.

HTML por sí solo no proporciona estilos atractivos para las tablas, por lo que se usa CSS para mejorar la apariencia.

  • border → Agrega bordes a la tabla.
  • border-collapse → Elimina espacios entre bordes.
  • padding y text-align → Ajustan la alineación y espaciado en las celdas.
  • background-color → Cambia los colores de fondo.
  • nth-child(even/odd) → Alterna colores en filas.
MDN Web Docs. (s.f.). HTML Tables. Mozilla. Recuperado el 15 de febrero de 2025, dehttps://developer.mozilla.org/en-US/docs/Learn/HTML/Tables

12. ¿Cuáles son los principales controles de una forma HTML5?

Controles básicos de entrada (<input>)

  • <input type="text"> → Campo de texto de una línea.
  • <input type="password"> → Campo de contraseña (oculta los caracteres).
  • <input type="email"> → Valida automáticamente correos electrónicos.

Controles de selección

Estos elementos permiten elegir opciones predefinidas.

  • <select> → Lista desplegable.
  • <option> → Opción dentro de <select>.
  • <input type="checkbox"> → Casillas de verificación (selección múltiple).
  • <input type="radio"> → Botones de opción (selección única).

Controles avanzados para mejorar la experiencia de usuario

  • <input type="file"> → Permite la carga de archivos.
  • <progress> → Barra de progreso.

Controles de texto avanzado

  • <textarea> → Campo de texto de múltiples líneas.
  • <input placeholder="Texto de ayuda"> → Texto guía dentro de los campos.
MDN Web Docs. (s.f.). Form controls in HTML5. Mozilla. Recuperado el 15 de febrero de 2025, dehttps://developer.mozilla.org/es/docs/Learn/Forms

=Sobre el ciclo de vida y desarrollo de los sistemas de información:=

¿Cuál es el ciclo de vida de los sistemas de información?

El Ciclo de Vida del Desarrollo de Sistemas de Información (SDLC System Development Life Cycle) es un proceso estructurado que define las fases necesarias para diseñar, desarrollar, implementar y mantener un sistema de información. Este ciclo garantiza que el sistema sea eficiente, seguro y alineado con los objetivos.

Fases

  • 1. Planificación (Planning): Se define el alcance, objetivos y viabilidad del sistema.
  • 2. Diseño del sistema: Se crea la arquitectura, interfaces y base de datos del sistema.
  • 3. Desarrollo o Implementación: Se programa el sistema según el diseño aprobado.
  • 4. Pruebas y Validación: Se verifican errores y se asegura que el sistema cumple los requisitos.
  • 5. Despliegue: Se despliega el sistema en producción y se capacita a los usuarios.
  • 6. Mantenimiento y Actualización: Se corrigen errores, se actualiza y optimiza el sistema continuamente.
    • Amazon Web Services (AWS). (s.f.). ¿Qué es el ciclo de vida del desarrollo de software (SDLC)?. Recuperado el 15 de febrero de 2025, de https://aws.amazon.com/es/what-is/sdlc/

15. ¿Cuál es el ciclo de desarrollo de sistemas de información?

El Ciclo de Desarrollo de Sistemas de Información es un proceso estructurado que se basa en el Ciclo de Vida del Desarrollo de Software (SDLC) y puede seguir distintos modelos de desarrollo como Cascada, Ágil o Espiral.

Fases del Ciclo de Desarrollo de Sistemas de Información

  1. 1. Planificación: Se identifican problemas, objetivos y viabilidad del sistema.
  2. 2. Definición o análisis de requisitos: Se definen y documentan las necesidades del usuario.
  3. 3. Diseño: Se establece la arquitectura, interfaces y base de datos.
  4. 4. Desarrollo del software: Se codifican los módulos y funcionalidades del sistema.
  5. 5. Pruebas: Se verifican errores y se optimiza el sistema antes del lanzamiento.
  6. 6. Implementación y Despliegue: Se instala el sistema y se capacita a los usuarios.
  7. 7. Operaciones y mantenimiento: Se aplican correcciones, mejoras y actualizaciones continuas.
Ortega, K. (23 de diciembre de 2022). Las 7 etapas del ciclo de vida del Desarrollo de Software. Avenu Global. Recuperado el 15 de febrero de 2025, de https://www.avenuglobal.com/blog/cuales-son-las-etapas-del-ciclo-de-vida-del-desarrollo-de-software-etapas-del-sdlc
